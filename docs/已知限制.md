# 已知限制

CardForge Studio 当前版本优先完成可运行闭环，以下能力仍是基础版或预留接口。

## 存储

- 默认使用浏览器 localStorage 保存项目。
- Chromium localhost 可实验性保存/打开可见项目文件夹。
- Tauri 2 文件系统适配仍是预留接口，尚未接入 Rust 侧实现。
- 尚未接入 IndexedDB 或 SQLite 索引，超大项目搜索和资源库会受浏览器内存限制影响。

## 格式兼容

- Character Card V1/V2/V3 JSON 和基础 CHARX 资源包已支持导入导出。
- V2 PNG metadata 支持 tEXt / iTXt / zTXt 的 chara / Chara 文本块读取，导出默认写入 tEXt/chara，仍需要更多社区样本兼容测试。
- CHARX 目前是 CardForge 基础资源包兼容：可打包 V3/V2 JSON 备份和关联资源，也可导入包内角色与资源；更多社区高级资源包变体尚未完整支持。
- 重型卡会尽量识别正则、变量、脚本、前端 UI 和插件线索，但不承诺完整运行第三方生态。

## AI

- 只内置 OpenAI-compatible chat completions / 多模态 chat 调用路径。
- 图片生成支持 OpenAI-compatible images、Stable Diffusion WebUI / A1111 常见 txt2img 响应，以及 ComfyUI `/prompt` 队列提交、可选 `/history` 轮询和 `/view` 图片拉取。
- Anthropic、Gemini 等类型目前用于配置和路由声明，完整原生协议仍待扩展。ComfyUI 轮询依赖用户在 Provider headers 中显式配置轮询次数和间隔；未配置或超时时仍只保存 `prompt_id` 任务记录。
- `proxy / relay URL` 需要用户自备 HTTP 转发端点。
- 费用估算基于 token 粗略计算，不等同于供应商真实账单。

## 高级模块

- Regex Lab、Variable Lab、Script Manager、Frontend Sandbox、Plugin Manager 已有基础管理和检查能力。
- 未知脚本和插件默认不执行。
- 项目级“完全信任”只允许管理界面标记脚本/插件启用；当前版本仍不会执行未知脚本或插件代码。
- 前端卡脚本只在受限 iframe sandbox 中预览，不提供宿主 API。
- MVU 仅支持常见 `_.set` / `_.assign` 命令解析；JSON Patch 和 JSON/YAML 状态块支持基础解析，尚未完整兼容所有社区语法。

## UI 与开发栈

- 当前使用 textarea 和手写 CSS，未接入 Monaco Editor、React Hook Form 或 Tailwind CSS。
- 手机端已有底部 Tab 与单栏适配，但复杂高级编辑仍更适合桌面宽屏。
- 端到端测试和截图验收仍需进一步扩展。
