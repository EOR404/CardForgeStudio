# 已知限制

- 当前不是 Tauri 桌面壳，Rust/Cargo 缺失时无法构建 Tauri。
- Chromium File System Access API 已支持实验性项目文件夹读写；非 Chromium 浏览器仍需使用 `.cardforge.zip` 或 JSON。
- `.cardforge.zip` 已能打包浏览器项目数据，但还不是 Tauri 原生文件夹工程的完整替代。
- localStorage 不适合超大图片库；大项目需要 IndexedDB 或文件系统。
- V2 PNG chunk 写入只覆盖常见 `tEXt/chara` 路径，需要更多样本验证。
- V3 JSON 已有实验性导入/导出；CHARX 和目标平台完整兼容尚未实现。
- 导出报告会列出目标平台风险和项目内依赖，但不能替代在 SillyTavern、Chub 等目标平台的实际试运行。
- 重型卡扫描为启发式识别，不等同于完整运行 SillyTavern 扩展生态。
- 资源库已有项目内用途和关联记录，但还没有跨项目全局资源库或移动到全局库。
- Regex、MVU、脚本、前端卡和插件模块是第一版基础能力。
- 不执行未知脚本，不运行未知插件。
- AI 调用日志已有 token 与费用粗估；未知模型价格表、图片生成 Provider 仍是后续增强项。
- A/B 对比目前覆盖角色快照与当前编辑状态的本地评分；多模型自动跑测、世界书版本、变量、脚本和整个项目的高级版本对比仍待增强。
