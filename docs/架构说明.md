# 架构说明

CardForge Studio 当前采用 React + TypeScript + Vite。

主要目录：

- `src/core/schema`：内部项目、角色、世界书、资源、AI、插件等类型和校验。
- `src/core/examples`：内置示例项目和模板数据。
- `src/core/importer`：V1/V2/世界书导入与兼容性报告。
  - `scan.ts`：重型卡扫描、字段完整度、依赖线索和建议模式。
- `src/core/exporter`：V1/V2/世界书导出、V2 PNG metadata、发布前检查和导出报告。
- `src/core/project`：`.cardforge.zip` 项目包导入导出。
  - `duplicate.ts`：复制项目数据并生成不冲突的项目名。
- `src/core/worldbook`：世界书触发模拟、变量条件判断、条目分类、AI 候选条目规范化与审计，输出命中、条件未满足、未命中、预算排除、插入位置/order 和 token 消耗。
- `src/core/ai`：OpenAI-compatible client、AI 任务封装与调用日志；支持自定义 headers、topP、stream 标记和 OpenAI-compatible SSE 文本合并。
  - `logging.ts`：记录任务类型、Provider、模型、token、耗时、错误、原始输出摘要和粗估费用。
  - `cost.ts`：汇总项目累计费用、今日费用和 Provider 成本对比。
  - `presets.ts`：AI 预设模板渲染、变量检查和任务预设选择。
- `src/core/prompt-builder`：测试沙盒 Prompt 构建，输出完整 Prompt 与分段链路、token、来源和跳转目标；传入项目上下文时会展示变量状态、前端初始化、脚本静态引用和 prompt 正则流水线。
- `src/core/tester`：内置测试用例、自定义测试用例运行入口、测试用例 Prompt 渲染、本地失败规则检查和角色快照 A/B 对比报告。
- `src/core/quality`：基础质量检查。
  - 世界书检查覆盖空内容、关键词缺失/过宽/重复、常驻预算、插入顺序、概率、depth 和变量条件配置。
- `src/core/search`：项目内存搜索索引，返回搜索结果、命中字段和页面跳转目标。
- `src/core/regex`、`src/core/variable-system`：高级实验室基础能力；Regex 支持流水线测试与错误回传，变量系统支持 MVU 命令识别、diff 预览、应用更新和历史快照。
- `src/core/frontend`：前端卡 sandbox 预览构建、按钮识别、资源绑定预览、初始消息/变量预览、世界书触发预览、AI 内容预览、权限白名单审计、Worker 预检脚本和 iframe 事件日志注入。
- `src/core/version`：角色与世界书快照 diff、快照数据读取与摘要；项目级快照由 store 保存/恢复并在导出时清洗嵌套 API Key 与敏感 header。
- `src/core/script-runtime`：脚本安全检查和运行时预留；当前版本固定不执行脚本。
- `src/core/plugin-runtime`：插件 manifest 规范化、安全检查和运行时预留；当前版本固定不执行插件。
- `src/plugins/api`、`src/plugins/examples`：插件 API 类型草案和示例 manifest/模块。
- `src/storage`：浏览器本地存储适配器，后续可替换为 TauriStorage。
  - `StorageAdapter.ts`：统一存储适配接口。
  - `BrowserFileSystemStorage.ts`：Chromium File System Access API 的项目文件夹读写。
  - `TauriStorage.ts`：Tauri 文件系统预留实现。
- `src/stores`：Zustand 全局状态。
- `src/pages`：应用页面。

数据策略：

- 第一版默认使用 localStorage 保存项目 JSON。
- 支持在 Chromium/localhost 环境中把项目保存为可见文件夹，目录包含 `project.json`、`characters/`、`worldbooks/`、`assets/`、`frontend/`、`scripts/`、`tests/test-cases.json`、`tests/failure-cases/`、`versions/characters/`、`versions/worldbooks/`、`logs/ai-calls.json` 等。
- 项目导出会清空 AI Provider 的 `apiKey`，并递归清洗 `Authorization`、`X-API-Key`、token、secret、password 等敏感 header/字段。
- 导入会保留原始 JSON 到 `originalSource`，并生成兼容性报告。
- 角色导出可以使用当前编辑状态，也可以从 `versions` 中的角色快照读取快照数据，并把 `characterVersionId` 写入导出历史。
- 资源记录会保留 `purpose`、`source`、标签、角色/世界书关联；导出页会优先推荐角色头像和导出封面用途的图片。
