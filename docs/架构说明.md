# CardForge Studio 架构说明

CardForge Studio 当前是 React + TypeScript + Vite 的本地优先 Web/PWA 基础版。核心业务逻辑放在 `src/core/`，页面只负责组合状态、交互和可视化。

## 目录分层

- `src/core/schema/`：项目、角色卡、世界书、资源、AI、测试、高级模块的集中类型、默认值和校验。
- `src/core/importer/`：V1/V2/V3/V2 PNG、世界书、复杂卡扫描，以及 Regex/脚本/前端卡高级资源导入。
- `src/core/exporter/`：V1/V2/V3/V2 PNG、世界书、导出报告和敏感信息清理。
- `src/core/worldbook/`：世界书分类、AI/本地文本条目规范化、触发模拟和预算排除分析。
- `src/core/ai/`：OpenAI-compatible 客户端、Provider/Preset 路由、调用日志、费用估算和 AI 操作。
- `src/core/prompt-builder/`：测试聊天 Prompt 构建和链路分段。
- `src/core/quality/`：角色、世界书和 token 分布检查。
- `src/core/regex/`、`src/core/variable-system/`、`src/core/script-runtime/`、`src/core/frontend/`、`src/core/plugin-runtime/`：高级工程模块的基础能力。
- `src/storage/`：浏览器 localStorage、可见项目文件夹和 Tauri 预留适配。
- `src/stores/`：Zustand 全局状态、项目编辑动作、撤销重做、导入备份和快照。
- `src/pages/`：项目、仪表盘、角色、世界书、资源、AI 设置、测试沙盒、导出和高级模块页面。

## 数据流

用户操作页面组件后调用 `useAppStore` 中的动作。动作修改 `CardProject` 数据，并通过 `BrowserStorage.save` 自动保存到浏览器本地存储。

导入流程会先进入 importer 和 normalize 层做结构校验、兼容扫描和字段补全，再写入项目。复杂卡会生成只读扫描报告，用户确认后才转换导入。

导出流程会从内部模型转换成目标格式，并通过 `sanitizeProjectForExport` 清空 API Key 和敏感 headers。V2 PNG 会把角色 JSON 写入 PNG metadata，同时保留 JSON 导出能力；导出页支持多图片批量生成 V2 PNG，并为每张图片写入独立导出记录。

项目级 `compatibilityTarget` 会记录默认发布目标，例如 SillyTavern 标准 V2、Regex、MVU、Chub、CardForge 原生或本地测试。导出页读取并更新该字段，发布前检查和依赖说明按同一目标计算。

AI 调用由 Provider 与 Preset 分离管理。任务路由顺序为：预设显式 Provider、Provider 默认任务类型、当前选中 Provider、首个可用 Provider。

## 安全边界

- API Key 只在本地编辑，不写死在代码里，项目导出时清空。
- 项目级 `trustLevel` 使用“不信任 / 受限信任 / 完全信任”三档策略，影响高级页开关和导出发布前检查。
- 未知脚本、插件默认禁用，当前版本只做静态管理和安全审计。
- 前端卡在 iframe sandbox 中预览，脚本执行需要显式受限开关。
- 导入和 AI 应用不会直接覆盖已有数据，用户需要确认。

## 存储策略

当前默认使用浏览器 localStorage。Chromium 环境下可实验性保存/打开可见项目文件夹，生成 `.cardforge` 结构化文件树。`TauriStorage` 已保留接口，后续可接入 Tauri 2 文件系统。
