# 插件开发草案

第一版只实现插件 manifest 管理、权限展示和只读 API 类型草案，不执行插件代码。

可在“高级 → Plugin Manager”新增插件草案，或导入 `.json` manifest。导入后会默认保留 `trusted: false`，并在界面中提示高风险权限。

示例 manifest：

```json
{
  "id": "author.regex-lab",
  "name": "Regex Lab",
  "version": "1.0.0",
  "main": "dist/index.js",
  "enabled": false,
  "trusted": false,
  "permissions": ["project.read", "regex.read", "regex.write"],
  "contributes": {
    "panels": [
      { "id": "regexLab", "title": "Regex Lab", "location": "rightPanel" }
    ]
  }
}
```

插件模块类型草案见 `src/plugins/api/types.ts`，示例文件见 `src/plugins/examples/`。

禁止默认开放：

- `apiKey.read`
- `fs.fullAccess`
- `shell.execute`
- `network.fullAccess`

后续插件运行时应放在受限沙盒中，并通过最小权限 API 访问项目数据。
