# 插件开发草案

CardForge Studio 当前版本提供插件 manifest 管理、权限审计、示例插件和类型草案，但不执行第三方插件代码。插件运行时会在后续版本中接入沙盒。

## 插件目标

插件可用于扩展：

- 导入器
- 导出器
- 面板
- 命令
- AI Provider
- 图片 Provider
- 变量解析器
- 脚本适配器
- 前端卡运行时
- 社区格式兼容器

## Manifest 示例

```json
{
  "id": "author.example-plugin",
  "name": "Example Plugin",
  "version": "0.1.0",
  "main": "dist/index.js",
  "enabled": false,
  "trusted": false,
  "permissions": ["project.read"],
  "contributes": {
    "panels": [
      {
        "id": "example",
        "title": "Example Panel",
        "location": "advanced"
      }
    ],
    "commands": [
      {
        "id": "example.command",
        "title": "Run Example"
      }
    ]
  }
}
```

## 权限原则

安全权限示例：

- `project.read`
- `character.read`
- `worldbook.read`
- `regex.read`
- `variable.read`
- `asset.read`

高风险权限会被提示：

- `apiKey.read`
- `fs.fullAccess`
- `shell.execute`
- `network.fullAccess`

当前版本不会授予插件读取 API Key、访问文件系统或执行 shell 的能力。

## 当前实现范围

- `src/core/plugin-runtime/manifest.ts`：manifest 规范化、安全检查和运行状态判断。
- `src/plugins/api/types.ts`：插件 API 类型草案。
- `src/plugins/examples/`：示例插件 manifest 和代码草案。
- `AdvancedPage` 的 Plugin Manager：导入、编辑、启用、trusted 标记和权限风险提示。
