# CardForge Studio 用户使用说明

## 启动

安装依赖后运行：

```bash
corepack pnpm dev
```

构建发布包：

```bash
corepack pnpm build
```

## 基础制卡流程

1. 在“项目”页新建项目，选择“轻量制卡”或“高级工程”，并设置默认兼容目标。
2. 进入“角色卡”页，新建角色或导入 JSON/PNG/CHARX。
3. 编辑 `name`、`description`、`personality`、`scenario`、`first_mes`、`mes_example`、`creator_notes`、`extensions` 等字段。
4. 在右侧查看质量检查、token 估算和 V2 JSON 预览。
5. 到“资源”页导入头像或封面图片，并设为角色头像。
6. 到“测试”页发送消息，查看 Prompt 链路和世界书触发日志。
7. 到“导出”页选择角色、版本、图片、格式和目标平台，确认兼容检查后导出。图片可来自项目图片库、全局图片库、最近使用或临时导入文件；V2 PNG 可勾选多张图片批量导出，也可单独导出角色头像 PNG；CHARX 会打包角色 JSON、V2 备份和关联资源。

项目页的“内置示例项目”提供 8 条可直接创建的样例。每个示例都有“查看说明”和“下载说明”，说明里包含适合人群、操作路径和验收点，可作为第一次熟悉工作流的检查清单。

## 兼容目标

项目会保存默认兼容目标，可在“项目”页创建时选择，也可在“仪表盘”页随时调整。导出页默认使用该目标生成发布前检查、数据损失提示和依赖说明；在导出页切换目标会同步保存回当前项目。

## 项目信任等级

项目会保存信任等级，可在“项目”页创建时选择，也可在“仪表盘”页调整：

- 不信任：只读查看高级依赖，禁止开启脚本、插件和前端 JS 预览。
- 受限信任：允许前端卡在 iframe sandbox 中预览受限 JS，但脚本和插件仍不能启用。
- 完全信任：允许在管理界面标记脚本、插件和前端 JS 预览；当前版本仍不会执行未知脚本或插件代码。

导出页发布前检查会把信任等级纳入安全提示。

## 导入角色卡

角色页支持导入：

- TavernAI / Character Card V1 JSON
- Character Card V2 JSON
- Character Card V3 JSON
- 基础 CHARX 角色资源包
- V2 PNG metadata 角色卡

轻量卡会直接导入。检测到正则、变量、脚本、前端 UI 或复杂 extensions 的重型卡会先显示只读扫描报告，确认后才转换导入。

如果 JSON 文本解析失败，角色页会保留原文摘录和错误原因。配置好可用于 `jsonRepair` 的文本 Provider 后，可点击“AI 修复并导入 JSON”；修复结果必须先解析成角色卡，才会写入项目，失败时不会修改现有角色。

## 世界书

在“世界书”页可以创建世界书和条目，编辑关键词、二级关键词、常驻、选择性触发、插入位置、顺序、优先级和变量条件。

可以直接导入 `.txt` / `.md` 长设定文本，或把设定粘贴到右侧文本框后点击“本地拆分条目”。本地拆分不会调用 AI，也不会自动写入世界书；确认预览后再点击“应用”。

右侧触发测试会显示：

- 已插入条目
- 命中关键词
- 插入顺序和位置
- token 消耗
- 被预算排除或未触发的原因

## 测试沙盒

“测试”页会用当前角色、世界书、变量树和高级依赖构建 Prompt 链路，并显示每段来源、token 和可跳转位置。AI 回复中如果包含 MVU `_.set` / `_.assign`、JSON Patch 或 JSON/YAML 状态块，右侧会显示变量 diff；只有点击“应用到 Variable Lab”后才会写入变量树。

## AI 辅助

先在“AI”页添加 Provider，并配置 Base URL、API Key、模型名和参数。然后可在角色页执行：

- 一句话生成角色草稿
- 字段生成、润色、缩短、扩写、风格化
- 角色问题检查
- 导入失败 JSON 修复
- 从角色卡拆分世界书建议

AI 输出会先显示原文和解析结果，点击“应用”后才写入项目。

## AI 图片生成

在“资源”页可以使用当前角色生成图片：

1. 在 AI 设置页配置可用于 `imagePrompt` 的文本 Provider，以及可用于 `imageGeneration` 的图片 Provider。
2. 回到资源页，选择当前角色、图片用途和风格。
3. 点击“生成提示词”，得到 positive / negative prompt。
4. 手动调整 prompt、尺寸、steps 和候选数量。
5. 点击“生成图片”。
6. 候选图只会显示在页面中，点击“保存到资源库”后才会写入项目，并自动关联当前角色。

当前图片生成支持 OpenAI-compatible images 返回的 `b64_json` / `url`，Stable Diffusion WebUI / A1111 的 `images` base64 返回，以及 ComfyUI `/prompt` 队列提交。ComfyUI Provider 如果配置了轮询 headers，会自动查询 `/history/{prompt_id}` 并拉取 `/view` 候选图；未配置轮询或超时未完成时会显示任务记录，可保存到资源库，最终图片仍可在 ComfyUI 侧查看或导出后再导入。

资源详情里可对当前图片执行“AI 分析图片”。该功能使用 `imageUnderstanding` 任务路由和支持 vision 的 Provider，返回描述、标签、用途建议和提示词线索；结果只会先显示在页面中，点击“应用分析到资源”后才会合并到 tags、purpose 和 notes。

## 格式转换

“导出”页侧栏的“格式转换”可把粘贴或导入的 JSON 转为 Character Card V1、V2、V3 或世界书 JSON。也可以从当前选择的角色/世界书载入源 JSON。转换结果只会预览和下载，不会自动写入项目；V1 转换会显示不支持字段的数据损失提示。

## 项目资源库与全局资源库

“资源”页上方列表是当前项目资源库，会随项目 JSON、项目包和导出记录保存。资源详情中的“复制到全局库”会把当前资源复制到本机全局资源库，并清除角色/世界书关联。

全局资源库显示在项目资源列表下方，适合保存常用头像、参考图、封面、脚本或文本素材。全局资源不会进入单个项目导出；点击“复制到项目库”后，会生成一个新的项目资源，随后才会随当前项目保存和导出。

如果同时有多个项目，资源详情里会显示“目标项目”和“复制到其他项目”。跨项目复制会在目标项目生成独立资源，并清除原项目中的角色/世界书关联，避免无效 ID 被带到另一个项目。

导出页也会列出全局图片库。点击“复制并选中”会先把全局图片复制成当前项目资源，再作为本次 PNG/头像导出的选中图片，并写入导出历史。最近使用图片来自当前项目的导出历史，会自动跳过已删除资源和非图片资源。

## 版本与恢复

角色页和世界书页都可以创建快照、查看差异并从快照恢复。高级页会按当前模块保存 Regex、变量、脚本、前端卡、插件或全部高级配置快照，恢复时只替换对应模块。项目导入角色或世界书前会自动创建项目备份快照。顶部“保存”会手动保存当前项目；普通编辑也会自动保存到浏览器本地存储。

## 高级工程

切换到“高级工程”后可进入：

- Regex Lab：测试正则匹配、替换和流水线。
- Variable Lab：编辑变量 JSON，解析 MVU `_.set` / `_.assign`、JSON Patch、JSON/YAML 状态块并预览 diff。
- Script Manager：管理或导入 STscript、Quick Reply、JavaScript、MVU 和模板脚本草案；导入后默认禁用。
- Frontend Sandbox：导入 HTML/CSS/JS/manifest 多文件，静态预览前端卡，查看权限和沙盒日志。
- Plugin Manager：导入或编辑插件 manifest，审计权限和 trusted 状态。
