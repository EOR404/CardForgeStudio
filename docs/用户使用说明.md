# 用户使用说明

1. 在“项目”页新建项目。
   - 可以导出 `.cardforge.zip` 项目包，用于完整备份和迁移。
   - 项目包和可见项目文件夹会额外写出 `logs/ai-calls.json`，便于审计 AI 调用历史。
   - 导入入口支持项目 JSON 和 `.cardforge.zip`。
   - 项目、角色或世界书导入失败时会在页面内显示错误，现有项目数据不会被覆盖。
   - 项目 JSON、项目包和项目文件夹会先检查是否像 CardForge 项目；浏览器本地存储主数据损坏时会尝试恢复上一份保存副本。
   - 在 Chrome/Edge localhost 中可以使用“保存到项目文件夹”和“打开项目文件夹”。
   - 可以从 8 个内置示例项目创建工程。
   - 项目列表支持复制项目，适合在大改前保留一份本地副本。
   - 打开项目后，可在“仪表盘”使用全项目搜索并跳转到来源页面。
   - 仪表盘可创建项目快照，并从快照恢复项目内容；恢复时会保留快照历史。
   - 顶部栏可以在“轻量 / 高级”之间切换当前项目模式；轻量模式会收起高级工程入口，高级模式会显示“高级 / 更多”入口。
   - 数据变更会自动写入浏览器本地存储，也可以点击顶部保存按钮手动保存并查看保存状态。
   - 顶部栏的撤销/重做按钮可回退最近的数据变更；只切换页面不会写入撤销历史。
2. 进入“角色卡”页，新建或导入 V1/V2 JSON。
3. 编辑角色字段，右侧会显示质量检查和 token 估算。
   - 导入复杂卡后，可在“高级 → Import Reports”查看扫描报告。
   - 报告会提示轻量/中型/重型卡、字段完整度、依赖线索和建议模式。
   - 所选角色的右侧“兼容性提示”会直接显示最近一次导入报告、依赖标签、只读原因和建议模式，并可跳转到高级报告。
   - 如果导入卡的 extensions 中包含 MVU/变量、正则、脚本或前端 HTML/CSS/JS 片段，会自动合并变量状态到 Variable Lab，生成 Regex Lab 规则、Script Manager / Frontend Sandbox 草案并挂到该角色；脚本默认禁用。
   - 角色页右侧可以创建带备注的版本快照，查看字段差异并从快照恢复。
4. 在“世界书”页新建条目，或粘贴长设定文本让 AI 抽取条目；条目会带分类，可用分类视图筛选，并用触发测试确认关键词和变量条件是否命中、插入顺序、位置、token 消耗、预算排除和未触发原因。
   - 问题检查会提示空内容、无关键词、关键词过宽或重复、常驻预算过高、插入顺序冲突、概率或 depth 配置异常。
   - 变量条件每行一条，例如 `quests.lostLamp.state == 进行中` 或 `player.hp >= 1`，会读取高级模块中的变量树。
   - 右侧可创建世界书快照，查看条目/预算等差异，并从快照恢复当前世界书。
5. 在“资源”页拖拽或选择导入图片、JSON、文本、脚本等资源。
   - 图片可标记为头像、导出封面、背景、表情差分等用途。
   - 可关联角色和世界书，并查看角色头像、世界书关联、导出历史等使用记录。
6. 在“AI 设置”页添加 OpenAI-compatible Provider，可配置模型列表、自定义 headers、topP、stream 和能力标签。
   - API Key 默认以密码框显示；如果自定义 headers 里包含 `Authorization`、`X-API-Key`、token、secret 或 password 类字段，页面会提示，并可一键清除 Key/敏感 headers。
7. 在“AI 设置”页编辑 AI 预设，可为不同任务绑定不同 Provider 或模型。
   - Provider 连接测试和实际 AI 调用会进入“AI 调用日志”，可查看 token、耗时、成功/失败、今日费用、累计费用和 Provider 成本对比。
8. 在“角色卡”页使用 AI 生成草稿、执行字段生成/润色/缩短/扩写/风格化、检查角色问题或拆分世界书建议。
9. 在“测试”页选择内置测试用例、维护项目自定义测试用例，或发送自定义消息，查看 Prompt 链路、完整 Prompt 和世界书触发日志，也可以点击“诊断”生成测试报告。
   - Prompt 链路会展示每段来源、插入位置、token 粗估、启用状态，并支持复制和跳转到来源页面；绑定变量系统、前端卡、脚本或 prompt 正则后，也会显示对应链路段和正则流水线预览。
   - 自定义测试用例会随项目保存，项目包和项目文件夹会包含 `tests/test-cases.json`。
   - 可把当前对话标记为失败样本，并填写失败备注；测试记录会保留用例、状态、Prompt、触发条目和诊断结果。
   - 创建过角色快照后，可以在 A/B 对比中选择快照 A，与当前编辑状态 B 对比评分、token、世界书触发、字段差异和修复建议。
10. 在“导出”页选择当前编辑状态或角色快照、目标平台、图片和格式，查看发布前检查与依赖说明，再导出 V1/V2/V3 JSON、世界书 JSON 或 V2 PNG。
    - 可下载 Markdown 导出报告，记录阻塞项、警告、目标平台和依赖说明。
    - 如果发布前检查存在阻塞项、兼容风险或 V1 数据损失提示，导出前会要求确认；确认内容会写入导出历史。
11. 在“高级 → Regex Lab”可新增、选择、编辑、删除正则规则，并查看按顺序执行的命中与替换结果。
12. 在“高级 → Variable Lab”可编辑变量 JSON，预览手动 diff，粘贴 `_.set()` / `_.assign()` 后应用变量更新，并从历史快照回滚。
13. 在“高级 → Script Manager”可维护脚本草案，编辑语言、触发器、权限和兼容性；危险权限会标红，脚本不会被自动执行。
14. 在“高级 → Frontend Sandbox”可编辑前端卡 HTML/CSS/JS，绑定资源库资源，用受限 iframe 预览，并查看初始消息、变量预览、按钮标签、世界书命中、发送给 AI 的内容预览、权限白名单、Web Worker 预检和沙盒事件日志；导出项目包或项目文件夹时，绑定资源会进入 `frontend/assets/`。
15. 在“高级 → Plugin Manager”可新增或导入插件 manifest，编辑入口、权限、contributes、trusted/enabled 状态；危险权限会标红，插件不会被自动执行。

AI 输出不会自动覆盖现有字段，必须点击“应用”。
